# 0.2.16 — 2026-05-22

## 重大变更：配置格式升级（无向后兼容）

### collect_mistakes / auto_collect_solution 改为 mapping

旧格式（string/bool）不再支持。acm-config 启动时强制检测并升级。

```yaml
# 旧
collect_mistakes: auto
auto_collect_solution: true

# 新
collect_mistakes:
  mode: auto
  perm: false
auto_collect_solution:
  mode: true
  perm: false
```

`perm` 字段跟踪对应文件的 Write 权限是否已在 settings.local.json 中配置。运行时权限缺失会警告。

### 配置项固定排序

acm-config 写入时按 acm/SKILL.md 解析顺序排列字段，减少模型跳读。

### 新增 `acm-data` skill — 导入导出

- 导出：`mistakes.md` + `solutions/` → `acm-export/` 文件夹
- 导入：从外部文件夹合并错误记录和题解（去重）

### acm-config 增强

- 版本检查增加**格式检测**：即使 config_version 最新，旧格式字段也会被识别并强制升级
- 配置错误收集/收录题解后自动跟进权限配置
- 新增"Permission Follow-up"流程

### acm/SKILL.md 增强

- 配置解析段重写：字段按顺序排列，显式标注预期格式
- 新增 Permission Warnings 段：缺权限时警告
- 移除所有 backward compat 说明

### 交叉引用同步

- acm/SKILL.md：配置解析 + perm 检查
- acm-config/SKILL.md：格式检测 + 排序输出 + 自动配权限
- acm-setup/SKILL.md：产出新 mapping 格式
- acm/references/code-review.md：mode/perm 引用更新
- skills/acm-data/SKILL.md：新建
