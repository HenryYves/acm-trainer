# 0.2.13 — 模板摘要拆分为独立文件 + 重新分析模板 diff 对比

## 背景
- 原设计：模板摘要写在 `.claude/acm-trainer.local.md` 的 markdown body 中，每次启动 acm skill 都会加载。
- 问题：纯算法讲解/题解场景不需要模板信息，白白占用 token。
- 用户反馈：`acm-config` 重新分析模板时说"无变化"，实际模板已变动（rd 容器/迭代器读取、Printer 迭代器输出），未做逐段 diff。

## 变更

### 模板摘要拆分
- **acm/SKILL.md**: Template-Aware Code Review 从读 config body 改为读 `.claude/acm-trainer/template-summary.md`。新增启动检查：`has_template=true` 但文件缺失时提醒用户运行 acm-config。
- **acm-setup/SKILL.md** Step 11: 模板摘要写入独立文件 `template-summary.md`，config body 只写一行标记。
- **acm-config/SKILL.md** Step 5: 同上。Step 3: 重新分析模板必须 diff 对比旧摘要。
- **code-review.md**: gotchas 来源改为 `template-summary.md`。
- **MODIFICATION.md**: 文件清单加 `template-summary.md`。

### 重新分析模板修复
- **acm-config/SKILL.md** Step 3: "重新分析模板"新增要求——分析完后读旧的 `template-summary.md`，逐段对比（别名/宏/IO/gotchas/常量），展示 diff 后再让用户确认。

## 教训
- 模板分析不是静态信息，每次重分析必须逐段对比旧摘要，不能假设"无变化"。
- 用户配置文件（`.claude/...`）和插件源文件（`skills/...`）要分清楚：改插件改源目录，用户侧由 setup/config 流程自动生成。
