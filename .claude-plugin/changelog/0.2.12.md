# 0.2.12

## 移除 duipai_exe_paths / 对拍

**日期**: 2026-05-13

**背景**: `duipai_exe_paths` 被误解为需要单独的"正解 exe"配置。原始设计意图是 `exe_paths` 本身就支持 bash 直接运行用户编译的 exe 验证 hack 数据。对拍功能（两 exe 对比）过度设计，用户不需要。

**改动**:
- acm/SKILL.md: 删除 `duipai_exe_paths` 字段解析 + Duipai 章节 + Quick Reference 行
- acm-setup/SKILL.md: 删除 Step 2c（对拍 exe 路径）+ 预览行 + YAML 模板字段
- acm-config/SKILL.md: 删除显示行 + 版本升级表条目 + 修改选项 + Step 3 处理逻辑
- code-review.md: 删除 Duipai 章节
- MODIFICATION.md: 删除配置项清单中的 duipai_exe_paths 行
- config_version → 0.2.10（表示 schema 变更：删除了 duipai_exe_paths 字段）

**向后兼容**: 旧配置中 `duipai_exe_paths: {}` 被忽略，不影响功能。

## 新增 Windows NT 退出码完整参考

**日期**: 2026-05-13

**背景**: 原退出码表只覆盖 3 个 Windows 码。用户整理了 11 个 NTSTATUS 码的完整对照表。这些不是 Agent 默认常识 — 模型无法从裸有符号十进制数反向映射到错误语义。

**改动**:
- acm/SKILL.md: 内联表扩充为 6 个 Windows 码 + Linux 信号，分为两个子表，新增 `0xC000001D` (ILLEGAL_INSTRUCTION)、`0xC0000409` (STACK_BUFFER_OVERRUN/GS)、`0x80070005` (ACCESS_DENIED)
- 新增 references/windows-exit-codes.md: 11 个 NTSTATUS 码的完整三进制对照表（hex/unsigned/signed），含 ACM 场景相关性和 bash 报告形式的说明
- MODIFICATION.md: 文件清单新增该参考文件

**设计决策**: 内联表保留高频码（代码 bug），低频码（环境/安装问题）只在参考文件中。遇到未知退出码时，skill 指引加载参考文件查表。
