# 0.2.4 修正详情（2026-05-10）

## 触发背景

1. **Code Location 段仍在触发文件搜索**：0.2.3 修了配置文件查找（Startup 段不再提 Glob），但 Code Location 段没有同步修复。模型从 `code_paths` 拿到确切路径后，又并行发起文件搜索（如 Glob "my*"）——和 0.2.3 完全相同的根因，不同的位置。
2. **acm-config 缺少权限配置选项**：acm-setup Step 13 加了权限配置（写 `.claude/settings.local.json`），但按照 MODIFICATION.md 交叉引用清单，acm-config 应同步新增对应选项——这一步漏了。从旧版本升级上来的用户无法通过 acm-config 配置权限。
3. **acm-config Step 2 选项过多**：8 个选项塞在一个 AskUserQuestion 中，超过 4 个上限，必报错。

## 修改内容

### acm/SKILL.md

1. **Code Location 段（L66-78）**：
   - 段首加硬约束：`The paths in code_paths are the exact file locations — use Read directly with the path from config. No lookup or verification is needed.`
   - 各 mode 的动词统一为"Read ... directly"，删除"look for""search"等词
   - `per_problem` 模式：删除"try common extensions"——路径由用户配置，不猜测
   - Quick Reference 表："Look up file via" → "Read the file at the exact path from"

### acm-config/SKILL.md

2. **Step 2 拆分**：原来 1 个 AskUserQuestion 塞 8 个选项 → 拆成 3 批（4+4+2），每批不超过 4 个
3. **新增"权限配置"选项**（Batch 2）：对应 acm-setup Step 13，写 `.claude/settings.local.json`
4. **Step 3 新增权限处理逻辑**：确认后读取/合并/写回 `settings.local.json`，与 acm-setup Step 13 逻辑一致

### acm-config/SKILL.md（追加）

6. **Step 1 新增版本对比**：展示当前配置版本 vs 最新版本（0.2.4），过旧时警告并建议运行 acm-setup

### acm-setup/SKILL.md

7. **Step 11 配置版本号**：`config_version: "0.2.0"` → `config_version: "0.2.4"`

### acm/SKILL.md（追加）

8. **版本检查阈值**：`older than "0.2.0"` → `older than "0.2.4"`，与 acm-setup/acm-config 保持一致

### MODIFICATION.md

9. 配置项清单 `config_version` 默认值 `"0.2.0"` → `"0.2.4"`

## 教训

- **0.2.3 的修复不完整**：只修了 Startup 段的 Glob 引用，没扫其他段是否也有同类问题。修改原则"不提工具名"应全局检查，非仅改一处。
- **交叉引用清单是强制清单，不是参考**：MODIFICATION.md 明确写了"新增/修改配置项时全部检查"6 个文件。acm-setup Step 13 加了权限配置后，acm-config 应立即同步——但当时没做。
- **改配置字段必须同步 bump config_version**：acm-setup Step 11 的 YAML 模板里硬编码了版本号。新增/删除配置字段时，`config_version` 必须同步升级，否则新旧配置无法区分。同时 acm/SKILL.md 和 acm-config/SKILL.md 的版本检查阈值也必须在同一批修改中更新。本次发现 0.2.1 加了 `auto_edit_code` 字段但版本号没动——这是漏了。
