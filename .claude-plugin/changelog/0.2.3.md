# 0.2.3 修正详情（2026-05-09）

## 触发背景

0.2.2 的 Startup 段写的是"**First** read... If not there, search with Glob..."——意图是先 Read 再 Glob。但实践证明模型看到"Glob"这个词就会在 Read 的同时并行发 Glob（当作"优化"），导致每次都多一次无意义的全局搜索。

## 修改内容

### acm/SKILL.md

1. **Startup 段（L12）**：
   - 改前：`**First**, read ... If the file is not there, search with Glob for **/acm-trainer.local.md within the working directory — do NOT search outside the project ...`
   - 改后：`**First**, use Read to read ... This file is always at this exact path after setup — do NOT search for it, do NOT use Glob. If the Read fails (file doesn't exist), stop and tell the user to run /acm-trainer:acm-setup first.`
   - **为什么这样写**：完全不提 Glob，不给模型"并行优化"的借口。配置文件经过 acm-setup 后永远在 `.claude/acm-trainer.local.md`，如果 Read 失败就说明没初始化，直接引导用户去 setup。

## 教训

- **指令里提到什么工具，模型就可能用那个工具**。即使写的是"if X fails then use Y"，模型也可能直接把 X 和 Y 并行。要阻止某个行为，最好的方式是根本不提它。
- 这次的问题和 0.2.2 的修复**根因相同但修复不够彻底**——0.2.2 加了"do NOT search outside project"，但保留了 Glob 这个词，所以没有真正解决问题。
