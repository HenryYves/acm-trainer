# 0.2.18 — 多 exe 路径 + perm 同步 + hack 格式 + 权限按需

## 背景

1. 用户可能用不同 IDE（VS2026、VSCode、Dev-C++）编译，exe 散落多个位置。之前一个关键词只能配一个 exe 路径，不同 IDE 编译后路径不同导致 hack 验证拿到旧的。
2. acm-config 的 `perm` 标记不会主动校验 settings.local.json 里真实的权限状态，导致手动加了权限后 `perm` 仍为 `false`。
3. Hack 数据格式加了中文标签（"输入:"、"期望:"）破坏直接复制。
4. 权限配置无条件加 Bash 权限，用户没配 exe 也加。

## 变更

### exe_paths 支持多路径（列表）
- `exe_paths` 每个关键词的值从**字符串**扩展为**字符串列表**
- 旧格式（单字符串）自动视为单元素列表，向后兼容
- 代码审查时：列出所有路径 → 找存在的 → 选最新的 → 验证新鲜度 → 跑
- acm-setup 自动寻找：找到多个时全部记录（不止选一个）
- acm-config 支持添加/移除/重新搜索单个关键词的路径

### perm 主动同步
- acm-config Step 1 每次都读取 settings.local.json 检查实际权限
- 若 `permissions.allow` 中已有对应 Write 权限 → 自动修正 `perm: true`

### hack 格式严格化
- 新增原则4: 数据块内禁止添加中文标签
- 正反例：`// 输入` 注释在外，数据块纯 raw

### 权限按需配置
- acm-config/acm-setup 权限配置不再无条件加 Bash(stat/exe)
- 仅在 `exe_paths` 非空时才加 Bash 权限
- Write 权限仍由 Permission Follow-up 单独控制
