# 0.2.5 修正详情（2026-05-10）

## 触发背景

用户报告两个问题：
1. acm skill 读取 `references/workflows.md` 等参考文件时弹出权限提示——权限配置只加了项目目录，没加插件缓存目录
2. 配置版本过期后 acm-config 提示"用 setup 更新"，但 setup 只有完全重新初始化——没有补全缺失配置的路径

在修复过程中用户进一步指出架构问题：配置版本号不应与插件版本号绑定；更新配置应由 acm-config 独揽，setup 只需提一嘴。

## 修改内容

### 架构决策

- **配置版本号 ≠ 插件版本号**：`config_version` 只在配置格式真正变化时才升。0.2.5 的配置格式与 0.2.4 完全相同，因此配置版本保持 `"0.2.4"`。
- **acm-config 独揽更新**：所有配置更新（补全缺失字段、修改选项、升级版本号）由 acm-config 全权负责。acm-setup 只负责首次初始化；当检测到旧配置时仅提一句"请用 acm-config 更新"。
- **新增 `remind_config_update` 字段**：用户可控制是否在配置版本落后时弹提醒。acm skill 启动时检查此开关，为 false 则跳过版本检查。

### acm-setup/SKILL.md

1. **Step 1**：撤销"补全"流程。配置存在时：显示摘要 → 如果版本旧则提一句用 acm-config → 问是否完全重新初始化。更新逻辑全部交给 acm-config。

2. **Step 11（YAML 模板）**：`config_version` 保持 `"0.2.4"`。新增 `remind_config_update: true`。

3. **Step 13（权限配置）**：`additionalDirectories` 新增插件缓存根目录（`acm-trainer` 目录），避免读 reference 文件弹授权。

### acm-config/SKILL.md

4. **版本检查**：完整重写为"对比 schema → 找出缺失 → 逐项引导配置"流程。补全后写入的 `config_version` 为 `"0.2.4"`（最新配置格式版本，非插件版本）。

5. **新增 `remind_config_update` 选项**：在 Batch 3 中可切换，Step 3 有对应处理逻辑。

6. **权限配置**：`additionalDirectories` 新增插件缓存根目录，`allow` 补 `Glob(**/*)`。

### acm/SKILL.md

7. **Startup 字段列表**：新增 `remind_config_update`（默认 `true`），`config_version` 默认值保持 `"0.2.4"`。

8. **版本检查**：改为 `if remind_config_update AND config_version < "0.2.4"` 才提醒，指向 acm-config。`remind_config_update` 为 false 时完全跳过。

## 设计原则

- **运行时缺字段** → 用默认值（不写盘），acm/SKILL.md 已有此行为
- **更新配置时缺字段** → 逐项让用户配置，不静默填入默认值
- **补全逻辑**：对比完整 schema，少什么补什么，不依赖版本间 changelog
- **配置版本 ≠ 插件版本**：配置格式不变就不升版本号
- **职责分离**：acm-config 管更新，acm-setup 管初始化
