# 0.2.10 修正详情（2026-05-12）

## 触发背景

1. 难题卡住后模型花 28 分钟无效思考，需要限轮退出 + 约束推断 + 收录系统三管齐下。
2. Admitting Difficulty 语气被指出"你这个价位"主语不当 + "非"太书面，改为模型自嘲口语。

## 修改内容

### skills/acm/SKILL.md

1. **Admitting Difficulty**：语气优化——"不是我这种价位的模型能想出来的"（主语改为模型自己，"不是"替代"非"）；触发条件具体化为"~3 轮推导无果或循环重复思路"。
2. **新增 `## Solution Collection` 章节**：两级收录系统（`index.md` 索引表 + `details/<slug>.md` 详情文件）。详情文件含"如何想到"引导段。收录前去重。支持 config `auto_collect_solution` 自动收录。
3. **新增 `auto_collect_solution` 配置项**：默认 `false`。用户可在 acm-setup/acm-config 中切换。
4. **Scenario Routing**：解题前先查 `index.md` 有无相关收录。
5. **Quick Reference**：新增"收录这道题"/"记录题解"条目。

### skills/acm/references/workflows.md

6. **Problem Solving Walkthrough**：新增 Step 0 查收录、Step 2 约束→复杂度→算法族推断、Step 7 三轮无果退出。

7. **新增 `## Constraint-Based Viability Check`**：正式扫描前先检查空间/时间复杂度是否从约束上就不可能（如 n≤1e9 时 `vector(n)` 直接判致命）。
