# 0.2.20 — Markdown auto 模式终端精简 + 追加内容双向锚点链接

## 背景

用户反馈 `markdown_output: auto` 模式下，终端和 .md 重复输出完整内容，浪费 token。同时 `.md` 内追加补充内容后没有链接，跳转不便。

## 变更

### 1. Markdown auto 模式：终端精简

`skills/acm/SKILL.md` § Markdown Output > Mode: auto

- 终端改为 **1–3 句摘要 + 文件路径**，不在终端和 .md 里重复全文
- 原因：终端输出留在会话历史里，每轮重读。终端精简 → 后续每轮省 token
- 例外：用户连续追问时正常对话，事后批量补写到 .md

### 2. 追加内容的双向锚点链接

`skills/acm/SKILL.md` § Mode: auto（Anchor links 小节）
`skills/acm/references/markdown-output.md` § 追加内容的双向链接

- 追加补充内容到 .md 时，在**原文章节**插入 `> 📝 补充：[标题](#锚点)` 前向链接
- 在**新章节末尾**插入 `> ⬆ 返回：[标题](#锚点)` 返回链接
- 锚点格式：Typora heading ID 或 `<a id="xxx">` 显式标签
- 修改流程：读文件→插入前向链接→追加内容+返回链接
