# 0.2.6 修正详情（2026-05-12）

## 触发背景

1. 用户说"代码在A中"，config 里 `code_paths.A` = `sub_project\A\A.cpp`，但模型却读了 `.\A.cpp`——把关键词当文件名拼接了路径，而不是从 `code_paths` 查映射。
2. 用户指出 `template_path` 字段分析完后再未使用（acm skill 只读 config markdown body 里的模板摘要），且模板未必在固定文件中——之前用户得临时把模板塞进一个文件才能做分析。

## 修改内容

### acm/SKILL.md

1. **Code Location 段 `files` 模式**：加显式反例——"Do NOT construct a path from the keyword name — `A` means `code_paths.A`, not `./A.cpp`"。删除原先容易跳过的粗体说明，改为在每种模式中直接写清查找规则。

2. **Startup 字段列表**：删除 `template_path`。`config_version` 默认值 → `"0.2.5"`。版本检查阈值 → `"0.2.5"`。

### acm-setup/SKILL.md

3. **Step 11 YAML 模板**：删除 `template_path` 行。`config_version` → `"0.2.5"`。

### acm-config/SKILL.md

4. **全局**：所有版本阈值 `0.2.4` → `0.2.5`。
5. **配置展示**：`模板代码: <路径 or 无>` → `<有/无>`。
6. **版本升级字段表**：删除 `template_path` 引用。

### MODIFICATION.md

7. **配置项清单**：删除 `template_path` 行。`config_version` 默认值 → `"0.2.5"`。
8. **依赖关系**：`读 template_path` → `分析模板代码（不记录模板路径）`。
9. **更新历史**：新增 0.2.6 条目。

### plugin.json

10. 版本 `0.2.5` → `0.2.6`。

## 设计决策

- **不记录模板路径**：`template_path` 只在 setup 分析时用到一次，acm skill 运行时只读 config markdown body 中的模板摘要。删除后旧配置中的 `template_path` 被忽略，不影响功能。
- **配置版本升级**：删除字段属于配置格式变化，因此 `config_version` 从 `"0.2.4"` 升到 `"0.2.5"`。与插件版本 `0.2.6` 独立。
