# 0.2.17 — 2026-05-26

## 行为修复：权限自动配置 + 收紧 + 拒绝记忆

### acm-config: Step 1 新增 Auto Permission Check

格式升级后自动检查 `collect_mistakes` / `auto_collect_solution` 是否需要写权限，需要则立即走 Permission Follow-up，再跟上通用读权限配置。之前格式升级完直接跳到 Step 2，权限配置被跳过。

**触发条件**：仅在格式升级时触发一次，日常进 Config 不重复弹。

### 权限收紧：移除宽泛规则

acm-config "权限配置" 和 acm-setup Step 13 之前写入三条宽泛规则：
- `Glob(**/*)` — 允许全局搜索，acm 本身禁用了 Glob，完全不需要
- `Bash(ls *)` / `Bash(dir *)` — 允许任意目录列表，不必要

`additionalDirectories`（项目目录 + 插件缓存根目录）已覆盖所有 Read 操作（配置文件、代码文件、reference 文件）。写权限按功能精准添加（`Write(.claude/acm-trainer/mistakes.md)` 等）。

### 拒绝即记录：不再反复询问

Permission Follow-up 中用户选"跳过"后，`perm: false` 视为用户明确不同意。同一次会话内不重复询问，下次 Config 运行时格式升级已完成也不会再弹。

### 交叉引用同步

- acm-config/SKILL.md: Step 1 + Auto Permission Check + 权限收紧 + Permission Follow-up
- acm-setup/SKILL.md: Step 13 权限收紧
- 用户侧 `.claude/settings.local.json`: 清理残留宽权限（建议）
