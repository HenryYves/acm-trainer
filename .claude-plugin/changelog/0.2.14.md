# 0.2.14 — acm-config 选项数修正 + 错误收集增强（计数/时间 + 强制执行）

## 背景
- **acm-config AskUserQuestion 报 Invalid tool parameters**：Step 2 的 Q1(5选项) 和 Q3(5选项) 超过工具限制（max 4），导致配置页面显示失败。
- **错误收集未自动执行**：code review 找到 bug 后，即使 `collect_mistakes=true`，模型也不主动写 mistakes.md。原指令写"用户确认后记录"，但用户开启了自动收集就意味着不想每次手动确认。
- **错误记录无复现统计**：只知道首次日期，不知道犯了多少次、最近什么时候再犯。

## 变更

### acm-config Step 2 选项拆分
- **acm-config/SKILL.md**: 3 个问题 → 4 个问题，每个 ≤4 选项：
  - Q1: 代码位置、引导方式、自动修改代码、术语风格 (4)
  - Q2: 收录题解、编程语言、评测机速度、权限配置 (4)
  - Q3: 重新分析模板、版本更新提醒、可执行文件路径、错误收集 (4)
  - Q4: 变值常量、以上都没有 (2)
- 问题文本改为唯一值（第 1/2/3/4 组 + 不同描述），避免 "Question texts must be unique" 错误。

### 错误收集增强
- **acm/SKILL.md Mistake Collection** + **code-review.md Mistake Collection** 同步更新：
  - 强化"必须执行"指令：找到 bug 后直接写入，不等用户确认。
  - 新格式：`[类别] 描述 — 次数: N, 首次: YYYY-MM-DD, 最近: YYYY-MM-DD`
  - 去重逻辑：同类别+同描述 → 更新次数和最近日期，不新增行。
  - 历史再现提示增加次数信息："共 N 次"。

### MODIFICATION.md 醒目警告
- 文件顶部加三级标题警告，强调"改源目录不改缓存"，防止第三次改错。

## 教训
- AskUserQuestion 的 options 上限是 4，设计配置页面时必须每个 question ≤4 项。
- 自动收集的含义是"自动"——用户打开开关后不应再弹出二次确认。强制执行指令用 **MUST** + 解释 why 效果比 may/can/should 好。
