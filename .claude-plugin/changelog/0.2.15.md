# 0.2.15 — collect_mistakes 三态枚举 + config_version 0.2.11

## 背景
- **错误收集只有开/关不够**：用户反馈即使开了自动收集，Agent 也不执行。问题有两层：(1) 原指令说"用户确认后记录"，与"自动"矛盾；(2) 只需要二态不足以表达用户意图——有人希望自动总结但看一眼再确认，有人希望完全静默。
- **错误描述绑定题面**：原来的描述（`ans=n 导致 n=3 时错误`）离开具体题目后无法理解，Agent 下次看到认不出同类错误。

## 变更

### collect_mistakes: bool → 三态枚举
- **acm/SKILL.md** Startup: 字段类型改为 string enum，默认 `"manual"`，向后兼容 `true`→`"auto"`。
- **acm/SKILL.md** Mistake Collection: 分三段描述 manual/confirm/auto 行为。
  - `manual`: 不自动，仅用户说"记录这个错误"时记录
  - `confirm`: 自动总结 → 列出条目 → 问用户"收不收" → 用户确认后写入
  - `auto`: 自动静默录入，不提确认
- **acm-setup/SKILL.md** Step 4c: 3 个选项。
- **acm-config/SKILL.md** Step 1/Step 3: 显示 + 修改均支持三态。
- **code-review.md**: 同步三态逻辑 + 新增"哨兵值设置错误"分类。

### 错误描述泛化
- 新增 **Good vs bad** 示例：❌ 绑定具体题面 vs ✅ 通用模式
- 分类新增 `哨兵值设置错误`（OI Wiki 术语）

### config_version: 0.2.10 → 0.2.11
- 配置字段 `collect_mistakes` 从 `bool` 变为 `"manual"|"confirm"|"auto"`，属于 schema 变更，version 必须升级。
- 同步更新 acm、acm-setup、acm-config 三处版本引用。

## 教训
- 用户说"自动"意思是"自动"——开关打开后不应再等确认。但如果有人想先看摘要再决定，需要中间一档 `confirm`。
- 错误模式描述必须脱离具体题面才可复用。好的描述 = 类别 + 通用问题 + 症状，不需要题干上下文就能判断。
